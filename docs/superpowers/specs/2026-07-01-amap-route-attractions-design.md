# 高德导航路线与沿线看点设计

## 目标

将行程人数从 7 人更正为 8 人，并升级路线图：每日轨迹使用高德驾车规划结果，地图同时支持行程精选看点和高德周边看点。规划失败时不绘制停靠点直线，避免将错误轨迹误认为真实道路。

## 问题根因

当前页面在初始化时通过 `Promise.all` 同时发起 7 次驾车规划。任一请求返回 `error` 或 `no_data` 时，代码会静默改用停靠点坐标折线，视觉上即表现为跨越地形的直线。后两日的问题是路线请求失败后的错误降级，而不是应当保留的路线样式。

## 路线规划

- 使用 `AMap.Driving` 和每日已有起点、途经点、终点获取导航路径。
- 当前选中日优先请求；其余日线路按队列顺序加载，避免同时发起 7 次规划。
- 成功路径在当次页面会话内按 Day ID 缓存，切换日期时复用。
- 暂时性错误自动重试一次；仍失败则只保留底图与停靠点，显示失败原因、“重试”和“在高德中打开”。
- 选中某日时，地图聚焦该日路线与标记，其他日路线低透明度显示。

## 沿线看点

### 行程精选层

- 默认开启。
- 来源为路书已确认的景点、摄影点、人文停靠点与风险点。
- 每个看点保存坐标、类型、建议停留时间、拍摄或游览建议和安全提醒。
- 地图标记使用黄色，点击后在右侧详情区展示完整信息。

### 高德周边层

- 提供独立开关，默认关闭，避免地图初次加载过度拥挤或产生额外请求。
- 开启后，仅针对当前选中日路线搜索景区、观景台和人文景点。
- 以路径上的有限采样点搜索周边 POI，对结果去重、过滤明显偏离路线的点，最多显示 8 个。
- 地图标记使用绿色，详情中明确标注“高德周边”，不与行程已确认安排混淆。

## 交互布局

- 保留现有左侧 Day 选择、中间地图、右侧详情结构。
- 在地图内增加“精选看点”和“高德周边”两个图层开关。
- 黄色标记表示行程精选，绿色标记表示高德周边，现有每日路线颜色继续用于导航线。
- 点击看点标记后，右侧当前停靠点区域切换为看点详情；切换 Day 时清空已选看点。
- 移动端将图层开关固定在地图顶部，看点详情在地图下方展示，不遮挡地图主体。

## 人数更正

- `tripMeta.people` 从 7 改为 8。
- 首页副标题与规模统计从“7 人”改为“8 人”。
- 更新对应测试和仅与当前产品数据直接相关的文案。

## 验证

- 测试人数为 8，两辆车信息保持不变。
- 测试路线请求按队列执行，选中日优先，成功结果被缓存。
- 测试高德返回 `error` 或 `no_data` 时不创建直线 `Polyline`，页面显示可重试状态。
- 测试精选看点默认显示，高德周边默认关闭，开启后结果去重且限制数量。
- 测试点击标记、切换 Day、路线重试和地图加载失败降级。
- 运行完整测试、TypeScript 检查和生产构建，并在桌面与移动视口截图验证地图不为空白、控件不重叠。
