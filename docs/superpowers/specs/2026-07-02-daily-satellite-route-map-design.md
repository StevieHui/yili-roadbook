# DAY6 路线约束与每日卫星路线图设计

## 目标

修复 DAY6 被高德按当前最短时间规划成西—东—西折返的问题，并在每日路书中展示当前日期的一张非交互卫星路线图。

## DAY6 路线约束

现有 `route` 同时承担可见停靠点和高德规划途经点。DAY6 在那拉提与乔尔玛之间没有独库公路控制点，高德可能根据当前道路状态绕开 G217。

为 `TripDay` 新增可选的 `routeControlPoints`。这些点只传给 `Driving.search`，不出现在停靠点列表、地图编号或路书文案中。DAY6 使用以下 GCJ-02 控制点：

- 独库公路那拉提入口段：`[84.340261, 43.308788]`
- 玉希莫勒盖隧道段：`[84.310378, 43.638115]`

规划顺序为：那拉提 → 两个独库控制点 → 乔尔玛路口 → 唐布拉 → 尼勒克。可见路线仍为那拉提 → 乔尔玛路口 → 唐布拉 → 尼勒克。

## 每日卫星路线图

新增独立的 `DailyRouteSnapshot` 组件，放在每日路书的当前日期内容中。组件只接收当前 `TripDay`，不创建七张地图。

地图使用高德 2D 地图与 `TileLayer.Satellite`，关闭拖拽、滚轮、双击、键盘、触摸缩放和全部控件。它加载当天真实驾车轨迹，绘制高对比轨迹线、编号停靠点及 `mapOnlyStops` 住宿标记。DAY7 缘居阁只显示为住宿标记，不进入机场还车前轨迹。

地图视野自动包含轨迹和标记。桌面采用 16:9 宽幅，手机采用更高的自适应比例，并设置不可交互的语义说明。

## 数据与代码复用

将驾车规划参数组装与路径搜索从大型交互地图中提取为共享 helper。交互路线图和每日卫星图使用同一规则：

1. 起点和终点来自 `day.route`。
2. 途经点按每段可见停靠点前插入对应 `routeControlPoints`。
3. `mapOnlyStops` 永远不参与驾车规划。

## 失败状态

高德加载或路径规划失败时，每日卫星图显示“当天路线暂未获取”及可见停靠点顺序，不绘制直线。现有每日路书的其余内容保持可用。

## 测试与验收

- 测试 DAY6 规划参数包含两个隐藏控制点且顺序正确。
- 测试控制点不出现在停靠点列表和可见标记中。
- 测试每日路书仅为当前日期渲染一张 `DailyRouteSnapshot`。
- 测试卫星图关闭交互、使用卫星图层，并在失败时显示停靠点顺序。
- 运行全量测试、类型检查和生产构建。
