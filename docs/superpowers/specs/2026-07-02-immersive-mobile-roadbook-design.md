# 沉浸式每日路书与全站移动端优化设计

## 目标

将现有七日动态路书升级为兼具沉浸感、设计感和执行效率的旅行页面。每天使用独立的纪实公路电影主视觉体现目的地地貌，同时确保路线、里程、驾驶时长和强度能在进入页面后立即读取。

本次范围同时包含全站移动端优化。主要验收宽度为 390px，并覆盖 360–430px 手机、768px 平板和 1440px 桌面。

## 产品结构

每日路书继续使用单个数据驱动页面，不创建七套重复组件。顶部保留七日日期标签，用于快速切换当天内容。切换后主视觉、主题强调色、路线指标、时间轴、摄影点和住宿信息同步更新，并将视图平滑返回当天开场顶部。

页面按以下三层组织：

1. **全屏景观开场**：高度约为 85–100vh，展示当天目的地景观。渐变遮罩上放置 DAY 编号、日期、路线、途经地，以及里程、驾驶时长和强度。
2. **行程执行区**：使用高对比度暖白背景，通过时间轴展示当天节奏。出发时间、关键驾驶节点和需要立即执行的信息获得更高视觉优先级。
3. **补充信息区**：摄影点、最佳拍摄时间、住宿信息采用编辑式分栏；手机端按使用优先级转为单列。

## 七日主视觉

图片由 AI 生成，统一采用真实摄影质感、自然光、克制电影调色，不包含文字、水印、品牌标识或不合理地貌。构图需为标题和指标预留稳定的低细节区域，并兼顾桌面横幅与手机竖向裁切。

## 信息层级与组件

### 日期导航

- 桌面端展示完整七日横向标签。
- 手机端使用可横向滚动的紧凑日期条，并让当前日期自动进入可视区域。
- 每个目标至少约 44px 高，具有明确的选中、悬停和键盘焦点状态。

### 景观开场

- 图片使用响应式资源与明确尺寸，避免布局偏移。
- 使用多层渐变保障标题、路线和指标的对比度，不依赖图片本身明暗。
- 首屏优先展示路线、里程、驾驶时长和强度；下沿露出进入时间轴的视觉提示。
- 手机端使用独立焦点位置，避免关键景物被标题或安全区域遮挡。

### 行程执行区

- 时间轴保持时间、活动、细节三级信息。
- 第一个出发节点与重要驾驶节点加强显示。
- 摄影点和住宿信息不与核心驾驶信息竞争首屏空间。
- 内容使用语义化标题、列表、时间元素和补充区域，保持键盘及屏幕阅读器可用。

## 全站移动端规则

- 顶部主导航改为紧凑的移动端菜单，避免导航项挤压或溢出。
- 多栏区块在手机端按任务优先级转为单列，而不是等比例缩小。
- 地图降低默认高度，并保留可读的路线摘要，保证弱网或地图加载失败时仍可执行行程。
- 日期条、标签组或宽表格仅允许组件内部滚动，页面本身不得出现横向滚动。
- 表单、清单、地图控件和按钮保持不小于约 44px 的触控区域。
- 统一调整移动端字号、行距、内容边距和固定导航占位。
- 打印样式继续隐藏交互和大图，保留七日行程、住宿、清单与关键提醒。
- 动效遵循 `prefers-reduced-motion`；低性能设备上不依赖视差或持续动画。

## 数据与状态

现有 `TripDay` 数据增加主视觉配置，包括图片路径、替代文本、焦点位置、主题强调色和景观标签。所有配置集中在行程数据中，组件只负责展示。

日期切换仍由现有 `selectedDayId` 驱动。切换后：

1. 更新当日内容与主题变量。
2. 将当前日期标签滚入可视区域。
3. 平滑返回当天景观开场；减少动态效果时改为立即跳转。
4. 预加载相邻日期图片，降低下一次切换等待。

## 图片失败与降级

- 当前主视觉加载失败时，显示基于当日主题色的渐变背景和景观标签，不显示破损图片图标。
- 相邻图片预加载失败不影响当前页面。
- 图片替代文本描述地点与地貌，不重复页面标题。
- 地图失败仍沿用现有静态路线摘要逻辑。

## Figma 与实现流程

先在 Figma 中建立桌面端 1440px 和手机端 390px 的核心每日路书画面，明确栅格、字号、间距、遮罩、图片焦点与组件状态。Figma 设计确认后，将设计还原到现有 React + TypeScript + 原生 CSS 架构，不引入新的 UI 组件库。

七张主视觉在实施阶段通过图片生成技能制作并进行实际裁切验证。生成图片不得覆盖现有通用封面图，文件使用明确的 Day 编号与地点命名。

## 测试与验收

- 1440px、768px、390px 及 360px 下无页面级横向溢出。
- 七日切换正确更新主视觉、指标、时间轴、摄影点和住宿。
- 当前日期在移动端日期条中始终可见。
- 主视觉加载失败时出现可读降级背景。
- 390px 下首屏清楚展示路线、里程、驾驶时长和强度。
- 导航、日期切换、清单和地图控件具备足够触控尺寸与键盘焦点。
- `prefers-reduced-motion` 下不执行平滑滚动或装饰动画。
- 打印预览不输出全屏背景图，并保留核心路书内容。
- 现有 Vitest 测试通过，新增日期切换与图片失败测试。
- TypeScript 检查与生产构建通过，浏览器控制台无错误。

## 非目标

- 不改写行程路线与地点内容。
- 不新增账号、后台、云端同步或图片管理系统。
- 不为七天创建重复页面或重复业务逻辑。
- 不引入重型动画框架或新的组件库。
